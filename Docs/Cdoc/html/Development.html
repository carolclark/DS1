Archived Projects and Repositories

Deprecated and Legacy: Status Notes

  • Accessor_C9
    • our current speech recognition application
    • started as a port from a CodeWarrior project; now in Carbon
    • to be replaced with a new Cocoa project and extensively refactored
  • Support_svn
    • predecessor to current project Support; features are currently being moved into project Support
  • DevSupport
    • older predecessor to current project Support; features are currently being moved into project Support
  • WebGen
    • This project was an attempt to capture techniques used by iWeb for use in a more structured way. It did not work, and now MobileMe, which provided support for some of the interesting features of iWeb sites, is no longer available.
    • Update our iWeb site to directly produce our Family subsite.
  • Koala Framework
    • These projects were developed to get started with building and testing Frameworks.
    • However, iOS requires self-contained apps, and therefore cannot use frameworks.
    • Koala_Try1
      • Framework project that also implements models for testing
      • Koala_Try1 builds the framework; does not have a separate build scheme, but is built by target KoalaUnitTests_Try1
      • KoalaUnitTests_Try1 builds the Koala framework and runs unit tests against it
      • AllTests_Try1 builds and runs tests in a standalone GHUnit application
      • Doxygen target builds a Doxygen docset
    • Koala_Try2
      • intended as a replacement for Koala_Try1, but only got as far as to work on revising the documentation
    • MyLibrary
      • partial implementation of a target that uses a custom framework
      • target MyLibrary: opens a standard window, and displays a log message to the console; uses framework Koala_Try1, but does not successfully #import it "Koala.h" header
  • ArbonneProto
    • attempt at implementing a replacement for the existing FileMaker Arbonne database in Cocoa and Core Data
    • demonstrates techniques we were using for Cocoa development
    • Core Data code was poorly structured, and probably should not be emulated
  • ArbonneFM
    • documentation for existing FileMaker Arbonne database; useful, at least for its specification of tables and fields, until a replacement has been developed
