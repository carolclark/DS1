ccInstall

CCDev supports an installation system used to process script-based targets. It is used primarily to install technical documentation, application configuration automation support, and general developer scripting support.

Install Scripts

Install scripts are files named <Name>_install.ksh, and are normally stored at the root level of their parent folder. Each script processes the files in its folder.

Install scripts are run in place, with the current directory set to their immediate parent folder. Any scripts or functions called should be invoked from their installed locations.

Arguments: Xcode's $SRCROOT; target name; action

code(s) for action(s) to be executed:

clean
trigger files for rebuilding; may perform additional cleanup
install
install files changed since last build, or all files if target was cleaned
test
execute shunit test files _Tests/test*.ksh associated with this target
upload
copy files from their installed locations to server

ccInstall Script

ccInstall provides support for install scripts. Install script template files are provided as part of our Xcode support.

Version Control

A Version Control System supports tracking the source files used in a project at multiple stages of development. It maintains a repository of the files, and updates that repository when the user requests that the files be "checked in". It normally supports viewing any two revisions of the same file side-by-side.

The Version Control System currently used at C & C software is Git, with repositories hosted at GitHub.

Permissions

Permissions Policy

In general, we use default permissions for most filetypes of rwxr-xr-x for directories and rw-r--r-- for files.

Files, directories and applications using different permissions:

  • Scripts and script templates, except Javascript: rwxr-xr-x
  • iWeb uses rwx------ for everything

Permissions Tasks

Add Group
  • check existing group IDs on the system to ensure unique ID
    dscl . -list Groups PrimaryGroupID
  • create group <gname>
    • sudo dscl . -create /Groups/<gname>
  • set group properties Password, PrimaryGroupID, RealName
    • sudo dscl . -create /Groups/<gname> <keyword> <value>
Add User
  • check existing user IDs on the system to ensure unique ID
    dscl . -list Users UniqueID
  • create user <uname>
    • sudo dscl . -create /Users/<uname>
  • set user properties Password, UniqueID, PrimaryGroupID, RealName
    • sudo dscl . -create /Users/<uname> <keyword> <value>
Add User to Group
