ThirdParty

CocoaPods effectively manages Cocoa software on which are products depend. ThirdParty is a project group for tracking and installing software in other languages on which our development depends.

Doxygen

	* in use for several years, but has a current installation bug
	* after bug is fixed, upgrade from 1.8.11 > 1.8.12, documenting process
	* also soon extending with bash-doxygen to generate docs for bash

Quex

	* lexical analysis for AccessorSlates, generated by this product,
	may begin the process of compiling Slates for new system to replace rez

Rez

	* our Slates have been compiled by Rez for many years using Build Cocoa Resources step in Xcode
	* Rez is deprecated by Xcode, and
	we could add more flexibility to our Slates if they had a more flexible format
	* will ultimately be replaced

Sharness

	* unix testing system that looked good at first
	* however, when we tried to get it up and running, we ran into serious problems,
	including dealing with Makefiles that appeared not to be Mac-compatible
	* we will be using bats

shunit2

	* we have been using this for several years, but it:
		* does not return a nonzero value on test failure
		* does not produce output compatible with TeamCity
		* xUnit style does not fit shell scripting well
		* appears not to be in active development
	* we will be using batsfor new development,
	and gradually transferring existing code

bashDB

	* appears to be best Bash debugger; source is huge
	* given TDD and the trace option, we may not need it; apparently many developers do not use it

ThirdParty Group Structure

This structure is designed to isolate cloned projects with their own repositories into their own folders, allowing for additional files to be placed in the outer folder. It differs from our normal project structure in that the repository's project is inside its folder rather than next to it.

	* Support
		* ThirdParty
			* <Product> folder
				* *placement inside folder allows for additional files, e.g. installers, to appear with product*
				* <Product>.xcodeproj
					* *contains cloned or forked repository from tool's host*
					* *placed inside folder to isolate code that belongs to another repository*
				* <Product>_install.sh
				* additional files as needed
			* ...
