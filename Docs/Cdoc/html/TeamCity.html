TeamCity

Installation and Configuration

  • install TeamCity application
    • Requires: Java, gnu-tar
      • check Supported Platforms and Environments; perform any support changes (e.g. update Java) needed
    • download from TeamCity downloads
    • Terminal:
      • cd to directory containing TeamCity archive
      • unzip using gnu-tar: gtar xf <archive>
    • move unzipped file to ~/Applications
  • Starting TeamCity Server
    • open http://localhost.8111; will not show content yet
    • ~/Applications/TeamCity/bin/runAll.sh start; localhost.8111 starts up
  • Configuration
    • user: carolclark
    • profile page: user name & email; UISettings: add "Show date/time in my timezone."
  • continue and document from Installing and Configuring the TeamCity Server
  • backup dsl configuration
  • Start TeamCity
    • open web page localhost:8111 (nothing there yet for new installation)
    • from TeamCity directory: bin/runAll.sh start
      • should show short log in Terminal, maybe beginning of TeamCity window at localhost:8111
  • Autostart TeamCity
    • copy jetbrains.teamcity.server.plist to /Library/LaunchDaemons
      • see Autostart TeamCity on Mac
      • restart computer
      • localhost:8111 should show TeamCity window
  • may take awhile for all the pieces to come together at this point;
    first time, no build agents appeared in evening, but were available next morning

Moving Projects to Another Server

  • make standard TeamCity backups of existing projects
  • see Project Import to import into new server
