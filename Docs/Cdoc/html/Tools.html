Apache

server With Subversion module

Installation and Configuration

Library /usr/local/apache; 2.2.19

Source MacPorts

License Apache Licenses

Installation and Configuration

  • Installation
    • install PHP via MacPorts > Available Ports
      • According to Stoyan Stefanov (author of "Object-Oriented JavaScript") in 2009 at Installing PHP and Apache on Mac OSX, Apache comes installed on a Mac, but you don't know what version you have. It's also disabled unless Web Sharing has been turned on in System Preferences. He recommends simply installing a current version of PHP5, which has apache2 as a dependency, through MacPorts, ignoring the versions of each that are already there. It appears to work well.
    • start apache, and set launch daemon on startup:
      sudo launchctl load -w /Library/LaunchDaemons/org.macports.apache2.plist
    • Subversion module
      • copy files mod_dav_svn.so and mod_authz_svn.so from /opt/subversion/lib/svn-apache to /opt/local/apache2/modules
        • sudo cp /opt/subversion/lib/svn-apache/*.so /opt/local/apache2/modules
  • Configuration
    • to edit Apache configuration file httpd.conf:
      sudo pico /opt/local/apache2/conf/httpd.conf
    • if desired, change the directory for the Web server host; see Change Host Directory
    • edit Apache configuration file httpd.conf to recognize Subversion and find repositories at <reposPath>
      • LoadModule: immediately after existing line to load mod_dav:
        LoadModule dav_svn_module modules/mod_dav_svn.so
      • after the LoadModule directives:
        • # Subversion Configuration
        • <Location /svn> DAV svn
          • SVNParentPath <reposPath>
        • </Location>
    • test
  • see also Support: Apache

PHP

scripting language used in web development, especially for server-side code

Installation and Configuration

Library /usr/local/bin/php, with related sibling files; php5

Source MacPorts

License PHP license; free

Installation and Configuration

  • PHP5 was installed with Apache in June 2011. It was not immediately used, and no configuration was completed. Configuration instructions are from the Mac ports installation.
  • tell Apache that php exists: sudo /opt/local/apache2/bin/apxs -a -e -n "php5" libphp5.so
    • this apparently adds the line LoadModule php5_module modules/libphp5.so to Apache's httpd.conf
  • configure PHP: choose one of the initialization files at /opt/local/etc/php5 (see comments in the files for more information); put a copy of chosen file in /opt/local/etc/php.ini
  • configure Apache to handle *.php files with php
    • in httpd.conf, add
      • AddHandler application/x-httpd-php .php
    • immediately after the line
      • LoadModule php5_module modules/libphp5.so
