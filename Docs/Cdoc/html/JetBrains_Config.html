TT_ConfigModel

TT_ConfigModel is a Team Tools project intended to help verify and maintain our customizations. Note that YouTrack, in particular, is tricky to get set up correctly, and does not provide for backup of its settings except for backup/restore of its entire database.

The process described here is intended to manage customizations of TeamTools configurations.

  • For any such customization for our development environment, the process is:
    • In this document, clearly define what and how changes are to be made. This is especially important because we may need to go back to reconstruct this configuration.
    • When one or more changes are ready to be developed:
      • For any change that can be copied between projects, e.g. Type and State values:
        • configure and test the changes in TT_ConfigModel
        • if copyable, copy into separate settings-only project TeamTools_CopySettings
      • otherwise, document and implement the process for making the change
      • Document, in JetBrainsAdmin.html, how to transfer the changes. This may be either by copying from TeamTools_CopySettings or by reproducing the steps.

In the associated Settings Table; planned customizations not yet implemented are colored as not yet implemented; customizations that have been implemented are colored as customized.

Note that any issues added to the developing testbed are in the connected GitHub project, and are version controlled.
