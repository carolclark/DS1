Development Overview

Agile Development

  • We use test-first agile development with BDD.
    • We are attempting to have our Agile boards provide for Behavior Driven Development.
    • Sprints have time estimates, not deadlines. Estimates are established based on card estimates, which are adjusted as indicated.
      • We expect to learn to make better estimates of time required as we use this technique.
  • We do not do deadlines.
  • Our Agile Boards
    • 'Roadmap' board for planning; 'Development' board for development
    • Swimlanes
      • named by Epic issues on Roadmap board, Story issues on Development board
      • those items are stationary until complete; they generate issues of other types to get the work done
    • BDD
      • Our boards are built to support Behavior Driven Development
      • Stories are orthogonal to Features, and may require functionality from otherwise unrelated Features.
    • Review
      • Since we cannot pass Review to another developer, we review our own work. We do try to come at each review with a fresh perspective, including some delay after completion.

Work in Progress

  • We are establishing WIP limits and related policies.
  • Tasks are broken down into smaller pieces as needed to conform to this policy.
  • For larger tasks as indicated, a required-dependency task is used to gather information and estimate.

DevOps: Automating Processes

A Process is a goal, with a sequence of steps intended to accomplish that goal. As processes and systems evolve, a Process may move gradually from manual to automated.

JetBrains Tools

available JetBrains IDE's for development languages:

  • IntelliJ IDEA: flagship development environment for Java and Kotlin
    • best support for related languages, project structure with modules, ...
    • includes at least part of several other IDEs; we have an all-products license for JetBrains IDEs
    • better choice for Java, javascript, bash, AppleScript
  • WebStorm: specializes in web development
    • we are currently using AppCode with sed(!!) for our documentation
  • AppCode (currently in use): primary support for Xcode projects associated with Cocoa/Objective-C(++)
    • plugins for unix and AppleScript
    • project structure based on Xcode, so no module support, limitations for other plug-ins and features
  • CLion: C-based languages, including C, C++, and Swift, not requiring build by Xcode
    • has some features for working with C++ and Swift together, including porting C++ to Swift
    • apparently, many developers are moving directly to Swift from C++, skipping Objective-C
  • other languages/IDEs
    • many of these are also available, at least at some level, from IntelliJ IDEA
    • PHPStorm for Website indices; DataGrip for internal databases; PyCharm with Django; MPS for DSL development
    • Kotlin for TeamCity scripting; javascript for Workflows

Team Tools for managing workflow and development process:

  • YouTrack: issue tracking; agile development
  • TeamCity (previously in use): Continuous Integration
  • UpSource: code review
  • Hub to tie it all together

We expect to use:

  • The complete team tool suite
  • IntelliJ IDEA: Java for kid coding; Kotlin for TeamCity scripting; Bash for shell utilities; javascript for workflows and new documentation process; by default for other languages
  • AppCode IDE for Cocoa projects, possibly moving to CLion/Swift
  • WebStorm for web development

JetBrains Adoption

Motivation

We have been using AppCode instead of Xcode for basic development for a little over two years (since Jan 2015). The decision to switch to AppCode was motivated by independent advocates, notably Jon Reid; their top reason was AppCode's refactoring capability in comparison with Xcode, with many other areas where AppCode was better.

About one year ago, we adopted JetBrains' TeamCity for continuous integration, and Kanban for planning through waffle.io.

Each of these steps helped us to improve our workflow considerably.

However, continuous integration frequently finds bugs for which we have a lot of difficulty finding a connection in the code. Most of these problems come from legacy code. We need better workflow to work more effectively on planning, implementation, and focus for add tasks, and especially with legacy code.

JetBrains also have tools for Developer and Manager workflow that integrate with each other, with TeamCity, and with the IDE. These include Agile planning boards with many advantages over waffle.io.

JetBrains is a large company with headquarters in St. Petersburg, Russia, and many other locations across Europe and the US. They also have a huge installed base of products, all of which receive many kudos for their quality. We do not expect them to disappear.

Team Tools

YouTrack: Issue tracker for development teams

JetBrains system for integrating issues. Links with GitHub. Replaces Waffle Kanban board with Agile boards (no scrums required) that also provide a specialized planning board, and provide integration with AppCode and other tools. Support a single set of Agile boards for all GitHub projects, allowing improved planning and prioritization without requiring combination of repositories.

TeamCity: Powerful Continuous Integration out of the box

Continuous Integration we have been comfortably using.

UpSource: Code review and repository browsing

Code review is normally done among peers, but we use UpSource to review code from our own multiple perspectives.

Hub: JetBrains Tools, integrated

Toolbox and IDEs

JetBrains provides a large variety of IDEs. We have been happily using their AppCode for Objective-C for a year now.

At some point, we will probably also be using other languages:

  • WebStorm for web development with javascript
  • PyCharm, Python with Django, for web development and text processing; also to handle existing (mostly unused) Python code
  • DataGrip for data management
  • PHPStorm to support a site-wide web searches
  • MSL, a DSL development system that could build a DSL for Slates to replace Rez for editing Accessor Slates

Toolbox is a JetBrains product that provides a control panel for access to all JetBrains IDEs, regardless of language. Toolbox would know which of our projects use which IDE, and would also support updating of JetBrains products.
