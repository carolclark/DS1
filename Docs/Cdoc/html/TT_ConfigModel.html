TT_ConfigModel

TT_ConfigModel is a Team Tools project intended to help verify and maintain our customizations. Note that YouTrack, in particular, does not provide for backup of its settings except for backup/restore of its entire database.

The process described here is intended to manage changes made to TeamTools configurations.

For any such customization for our development environment, the process is:

  • Clearly define what and how changes are to be made. This is especially important because we may need to go back to reconstruct this configuration.
  • When one or more changes are ready to be put into use:
    • For any change that can be copied between projects, e.g. Type and State values:
      • configure and test the changes in TT_ConfigModel
      • copy into separate settings-only project TeamTools_CopySettings
      • Transfer the changes, either by copying from TeamTools_CopySettings or by reproducing the steps.
    • Note that the issues testbed is in the connected GitHub project, and is version controlled.
